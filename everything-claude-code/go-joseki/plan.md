# 围棋定式学习网站开发计划

## 项目概述
使用 red/green TDD 开发围棋定式学习网站

---

## Phase 1: 棋盘基础功能

### 目标
实现可交互的围棋棋盘，支持落子、提子、悔棋等基础功能。

### 任务列表
- [x] 创建项目结构和技术栈配置 (Vite + React + TypeScript + Tailwind CSS)
- [x] 实现围棋棋盘渲染组件
  - 绘制 19x19 棋盘网格
  - 绘制星位标记
  - 响应式布局适配
- [x] 实现棋子渲染
  - 黑子、白子样式
  - 落子动画效果
- [x] 实现棋盘交互
  - 点击落子
  - 鼠标悬停预览
- [x] 实现围棋规则逻辑
  - 落子合法性检查
  - 提子规则
  - 劫争检测
- [x] 实现棋盘控制功能
  - 悔棋
  - 清空棋盘
  - 当前手数显示
  - 提子计数

### 技术要点
- 使用 HTML5 Canvas 绘制棋盘
- Zustand 管理棋盘状态
- 使用 TDD 开发核心逻辑 (goLogic.test.ts)

---

## Phase 2: 定式库

### 目标
构建可扩展的定式数据库，支持分类、筛选和查看。

### 任务列表
- [x] 设计定式数据结构
  - 定式名称、日文名
  - 类型（挂角、缔角、夹攻、点角等）
  - 分类（角部、边上、中腹）
  - 难度等级
  - 关键点说明
  - 标签系统
- [x] 实现定式数据
  - 星位定式 (hoshi)
  - 小目定式 (komoku)
  - 三三定式 (sansan)
  - 高目定式 (takamoku)
- [x] 实现定式列表组件
  - 卡片式布局
  - 难度星级显示
  - 类型标签
- [x] 实现定式筛选功能
  - 按分类筛选
  - 按类型筛选
  - 按难度筛选
  - 关键词搜索
- [x] 实现定式查看器
  - 棋谱展示
  - 步进控制（前进/后退/重置）
  - 当前步骤高亮

### 技术要点
- 使用 TypeScript interface 定义数据类型
- Zustand 管理定式状态
- 支持动态添加新定式

## 技术栈

- **框架**: React 19 + TypeScript
- **构建工具**: Vite 7
- **样式**: Tailwind CSS 4
- **状态管理**: Zustand
- **路由**: React Router 7
- **测试**: Vitest + React Testing Library
- **部署**: GitHub Pages

## 项目结构

```
go-joseki/
├── src/
│   ├── components/
│   │   ├── board/          # 棋盘组件
│   │   ├── joseki/         # 定式相关组件
│   │   ├── practice/       # 练习模式组件
│   │   ├── quiz/           # 测验组件
│   │   └── dashboard/      # 仪表板组件
│   ├── lib/
│   │   ├── goLogic.ts      # 围棋规则逻辑
│   │   ├── boardRenderer.ts # 棋盘渲染
│   │   ├── practiceEngine.ts # 练习引擎
│   │   └── quizEngine.ts   # 测验引擎
│   ├── store/
│   │   ├── boardStore.ts   # 棋盘状态
│   │   ├── josekiStore.ts  # 定式状态
│   │   └── progressStore.ts # 进度状态
│   ├── data/
│   │   └── joseki/         # 定式数据
│   ├── pages/              # 页面组件
│   └── types/              # 类型定义
├── public/                 # 静态资源
└── docs/                   # 文档
```
